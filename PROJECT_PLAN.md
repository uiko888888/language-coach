# Language Coach Project Plan

本文件是版本目标、范围、验收和发布状态的总账。它回答“这一版为什么做、做什么、不做什么、如何证明完成”。

## 产品北极星

用户把一份合法可用的真实内容交给 Language Coach 后，能够完成阅读、查词、词块积累、考试训练、证据复盘和后续复习，并清楚知道下一步应该练什么。

衡量产品进展时优先看：

1. 从素材进入有效训练需要多少步骤。
2. 错题能否回到明确原文证据并形成下一轮训练。
3. 推荐能否说明与用户目标、兴趣和弱项的关系。
4. 用户能否自由修改系统安排，而不是被推荐锁定。
5. 题目、释义和内容是否保留来源与质量边界。

## 版本工作流

每轮迭代按以下顺序执行：

```text
定义版本目标与非目标
-> 写明功能范围和验收标准
-> 实现与迁移
-> 自动化测试和浏览器验证
-> 更新 STATUS / ROADMAP / CHANGELOG
-> Conventional Commit
-> annotated Git tag
-> 用户明确要求后再推送 GitHub
```

需求变更分为四类：

- `新增`：加入当前或未来版本，并写明验收标准。
- `增强`：不改变功能边界，只提高效果、质量或效率。
- `延后`：保留价值，但当前依赖或优先级不足。
- `移除`：与产品定位、考试结构、版权或维护成本冲突。

## 当前迭代

### `v0.8.0-alpha.7` 个性化学习计划基础

**产品目标：** 用户打开应用时，能够看到一套符合自己可用时间和目标的今日学习组合，并可以自行修改。

**内容目标：** 今日文章继续来自可追溯来源和用户订阅；目标只影响排序与推荐理由，不修改原文，也不虚增考试能力分。

**范围：**

- 5 / 15 / 30 / 60 分钟每日计划。
- 阅读、做题、错题复盘、词块复习自由组合，至少选择一项。
- 近期目标、长期目标和目标日期。
- 用户可关闭画像推荐并退回通用内容安排。
- 今日内容接口返回计划和目标上下文。
- 设置保存在本地 SQLite，刷新和重启后保留。

**非目标：**

- 不实现系统级通知、浏览器推送或跨设备推送。
- 不声称已经完成动态能力画像或考试分数预测。
- 不使用目标文本直接改变能力分。
- 不在本版扩展新的考试题型。

**验收标准：**

- 每日计划至少选择一项任务，否则拒绝保存。
- 保存后再次读取的数据与用户设置一致。
- 今日内容返回同一份时长、任务和目标。
- 开启画像推荐时推荐理由说明“对应当前目标”；关闭后显示通用安排。
- 后端、前端语法和自动化测试通过。
- 更新版本文档、本地提交和标签；不自动推送 GitHub。

## 后续候选

### `v0.8.0-alpha.8` 训练历史与能力趋势

- 套题和单篇训练历史。
- 单次训练详情与重新进入。
- IELTS 能力维度的跨 session 趋势。
- 推荐下一步及可解释理由。

### `v0.8.0-alpha.9` 每日计划执行闭环

- 今日任务完成状态与剩余时间。
- 用户调整单项题量和词块数量。
- 将错题或插件素材加入今日计划。
- 计划完成后生成简短总结，不把 XP 当成能力分。

- 已有成绩和快速测试两条入口。
- 目标、兴趣、弱项、计划和推荐联动。
- 数据备份、恢复和核心端到端测试。
