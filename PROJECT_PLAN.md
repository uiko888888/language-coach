# Language Coach Project Plan

本文件是版本目标、范围、验收和发布状态的总账。它回答“这一版为什么做、做什么、不做什么、如何证明完成”。

## 产品北极星

用户把一份合法可用的真实内容交给 Language Coach 后，能够完成阅读、查词、词块积累、考试训练、证据复盘和后续复习，并清楚知道下一步应该练什么。

衡量产品进展时优先看：

1. 从素材进入有效训练需要多少步骤。
2. 错题能否回到明确原文证据并形成下一轮训练。
3. 推荐能否说明与用户目标、兴趣和弱项的关系。
4. 用户能否自由修改系统安排，而不是被推荐锁定。
5. 题目、释义和内容是否保留来源与质量边界。

## 版本工作流

每轮迭代按以下顺序执行：

```text
定义版本目标与非目标
-> 写明功能范围和验收标准
-> 实现与迁移
-> 自动化测试和浏览器验证
-> 更新 STATUS / ROADMAP / CHANGELOG
-> Conventional Commit
-> annotated Git tag
-> 用户明确要求后再推送 GitHub
```

需求变更分为四类：

- `新增`：加入当前或未来版本，并写明验收标准。
- `增强`：不改变功能边界，只提高效果、质量或效率。
- `延后`：保留价值，但当前依赖或优先级不足。
- `移除`：与产品定位、考试结构、版权或维护成本冲突。

## 当前迭代

### `v0.8.0-alpha.13` 双模式工作流与七天分项校准

**产品目标：** 兴趣与备考模式产生明显不同的学习路径，并让训练证据每七天以可解释、低波动的方式校准对应能力分项。

**内容目标：** 同一合法素材可进入沉浸阅读或考试训练，但模式不修改原文；分项训练不被夸大为综合英语或正式考试分数提升。

**范围：**

- 兴趣模式突出兴趣主题、订阅、沉浸阅读、词块和网页素材，隐藏未解决错题压力区。
- 备考模式展示目标、当前最低正确率能力、校准倒计时和弱项题型入口。
- 阅读、听力、词汇、写作和口语分项能力状态与校准历史。
- 每七天在完整训练归档后自动检查，按正确率、题目难度和信心错误微调。
- 单分项至少 8 题、2 个 session、跨 2 天；每周调整幅度有限。
- 综合等级需要至少 3 个有效分项，并同时含输入与输出能力证据。

**非目标：**

- 不把阅读进步外推为听力、口语、写作或综合等级进步。
- 不把 XP、打开文章、看过解析或计划完成度作为能力证据。
- 不输出官方 IELTS/TOEFL 分数预测或心理测量学认证。
- 不在本版加入单题耗时、提示使用、推荐反馈或拉片听力证据。
- 不推送 GitHub，除非用户明确要求。

**验收标准：**

- 两种模式返回不同模式契约、首页信号、主要动作和文章动作。
- 未满七天不运行校准；数据不足的分项保持不变并记录原因。
- 只提供合格阅读证据时仅阅读分项调整，综合等级保持原锚点。
- 校准记录算法版本、周期、证据、调整前后值和综合覆盖情况。
- 训练归档自动检查校准，不要求用户手动触发。
- 自动化测试、语法和差异检查通过；本地提交和标签，不自动推送 GitHub。

## 已完成迭代详情

### `v0.8.0-alpha.11.4` Windows 登录自动启动与部署边界

**完成范围：** 当前用户登录任务、延迟启动、单实例、补跑、失败重试、健康验证、状态检查、卸载和端口统一。

**验收结果：** 本机 `8766` 登录任务已注册，任务运行中且后端健康接口正常。

### `v0.8.0-alpha.11.3` 每日内容可靠更新

**产品目标：** 用户每天打开应用都能得到近期、可追溯且不会因单一来源失败而中断的内容更新。

**内容目标：** RSS 摘要仍明确标记摘要并保留原始链接；文章新鲜度使用真实发布时间，不使用导入时间伪装时效性。

**范围：** 启动异步刷新、运行中六小时调度、HTTP 缓存、三级去重、刷新日志、来源健康和指数退避。

**非目标：** 不在电脑关机或本地后端退出时运行；不抓取未经授权全文；不实现通知、事件聚类或自动停用来源。

**验收结果：** 76 项自动化测试、语法和差异检查通过；本地提交与标签完成。

### `v0.8.0-alpha.11.2` 私人 EPUB 分章导入

**完成范围：** EPUB 安全解析、私人书库、spine 分章、去重和按需送入阅读台。

**边界：** 原文件和正文只在本机 SQLite；不提供上传、分发、下载或版权判断。

### `v0.8.0-alpha.11.1` TOEFL 2026 Complete the Words 修正版

**完成范围：** 单空规则模拟、结构化掩码校验、错因、生词本回流和题目元数据持久化。

**边界：** `official_equivalence=false`；不是 ETS 官方多空界面、计时或评分复刻。

### `v0.8.0-alpha.11` TOEFL 高阶阅读题型

**完成范围：** 否定事实、修辞目的、句子插入和篇章总结单选基础，以及专属证据校验、错因与同类题。

**边界：** 总结仍为单选基础，不是 ETS 正式六选三；不含完整阅读套题和官方评分。

### `v0.8.0-alpha.10` 每日计划执行闭环

**产品目标：** 用户打开应用后不仅知道今天学什么，还能执行、调整和完成计划，并看到统一的实时进度。

**完成范围：** 单项数量目标、实时完成量与剩余时间、幂等素材队列、手动确认，以及做题、错题和新词卡的自动进度。

**边界：** 阅读不因打开页面自动完成；计划进度、XP 和能力趋势相互独立；不含推送、周报和跨设备同步。

### `v0.8.0-alpha.9` 训练历史与能力趋势

**产品目标：** 用户能够回看每次练习和模考的题目、答案与证据，并根据跨 session 表现知道下一步练什么。

**完成范围：** 训练归档、考试筛选、单次详情、能力/题型正确率、近期趋势、弱项建议、专项续练和证据回放。

**边界：** 趋势是近期描述信号，不是官方分数预测或知识追踪模型；不支持编辑历史答案、日期筛选、导出和同步。

### `v0.8.0-alpha.8` TOEFL 阅读专项基础

**产品目标：** 让 TOEFL 入口从“通用题换标签”升级为具有独立题型、校验和错因的可训练模块。

**内容目标：** 使用用户文章和合法来源素材生成 TOEFL 规则模拟题，保留原文、段落引用和 ETS 官方资源链接，不复制官方题目。

**范围：**

- 事实信息、推断、段落主旨、句子简化和语境词义五类独立模板。
- 四选一、证据追溯、段落引用、核心信息和目标词语境校验。
- TOEFL 专属能力、难度、生成来源和错因。
- 单篇混合训练、同题型巩固和现有训练/模考会话复用。

**非目标：**

- 不实现否定事实、修辞目的、句子插入、篇章总结和分类题。
- 不实现完整阅读套题、官方计时评分或听说写。
- 不导入或分发未经授权的 ETS 题目。

**验收标准：**

- 五类题型均由 `toefl-rule-v1` 生成且通过独立校验。
- 混合训练包含五类题型，不回落 `general-rule-v1`。
- 错题记录 TOEFL 专属错因，同类题保持原题型和 TOEFL 风格。
- 自动化测试、Python/JavaScript 语法和差异检查通过。
- 更新状态、研究边界、日志、本地提交和标签；不自动推送 GitHub。

### `v0.8.0-alpha.7` 个性化学习计划基础

**产品目标：** 用户打开应用时，能够看到一套符合自己可用时间和目标的今日学习组合，并可以自行修改。

**内容目标：** 今日文章继续来自可追溯来源和用户订阅；目标只影响排序与推荐理由，不修改原文，也不虚增考试能力分。

**范围：**

- 5 / 15 / 30 / 60 分钟每日计划。
- 阅读、做题、错题复盘、词块复习自由组合，至少选择一项。
- 近期目标、长期目标和目标日期。
- 用户可关闭画像推荐并退回通用内容安排。
- 今日内容接口返回计划和目标上下文。
- 设置保存在本地 SQLite，刷新和重启后保留。

**非目标：**

- 不实现系统级通知、浏览器推送或跨设备推送。
- 不声称已经完成动态能力画像或考试分数预测。
- 不使用目标文本直接改变能力分。
- 不在本版扩展新的考试题型。

**验收标准：**

- 每日计划至少选择一项任务，否则拒绝保存。
- 保存后再次读取的数据与用户设置一致。
- 今日内容返回同一份时长、任务和目标。
- 开启画像推荐时推荐理由说明“对应当前目标”；关闭后显示通用安排。
- 后端、前端语法和自动化测试通过。
- 更新版本文档、本地提交和标签；不自动推送 GitHub。
