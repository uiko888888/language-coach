## 产品北极星

用户把一份合法可用的真实内容交给 Language Coach 后，能够完成阅读、查词、词块积累、考试训练、证据复盘和后续复习，并清楚知道下一步应该练什么。

衡量产品进展时优先看：

1. 从素材进入有效训练需要多少步骤。
2. 错题能否回到明确原文证据并形成下一轮训练。
3. 推荐能否说明与用户目标、兴趣和弱项的关系。
4. 用户能否自由修改系统安排，而不是被推荐锁定。
5. 题目、释义和内容是否保留来源与质量边界。

## 版本工作流

每轮迭代按以下顺序执行：

```text
定义版本目标与非目标
-> 写明功能范围和验收标准
-> 实现与迁移
-> 自动化测试和浏览器验证
-> 更新 STATUS / ROADMAP / CHANGELOG
-> Conventional Commit
-> annotated Git tag
-> 用户明确要求后再推送 GitHub
```

需求变更分为四类：

- `新增`：加入当前或未来版本，并写明验收标准。
- `增强`：不改变功能边界，只提高效果、质量或效率。
- `延后`：保留价值，但当前依赖或优先级不足。
- `移除`：与产品定位、考试结构、版权或维护成本冲突。

## 当前迭代

### `v0.8.0-alpha.23.0` 统一记忆复习闭环

**产品目标：** 把生词本和已掌握错题从静态收藏升级为“到期、主动回忆、评分、重新排期、再次出现”的可执行记忆闭环。

**内容目标：** 复用用户已有单词、短语、原句和错题证据，不新增来源，不把重复浏览冒充有效复习。

**范围：**

- schema 7 统一复习项目与保留前后快照的非破坏性复习日志，覆盖单词、短语和已掌握错题。
- 独立调度器接口、四档评分、下一次到期时间、遗忘次数和复习状态。
- 到期队列、类型筛选、先回忆后显示答案、评分后的即时下一项和最近一次撤销。
- 每次有效评分进入今日复习进度；撤销同日评分时同步回退进度。
- 现有生词自动进入队列，新生词和新掌握错题自动注册。

**非目标：**

- 当前环境无法安装 `py-fsrs`，本版调度器明确标记为 `adaptive-interval-v1`，不宣称 FSRS。
- 不在没有真实复习日志时训练个性化参数，不根据一次自评直接修改语言能力画像。
- 不删除原生词本或错题教学路径，不改固定侧栏和左右主从布局。

**验收标准：**

- 老数据幂等迁移，队列只返回到期项目且不产生重复项目。
- 四档评分产生单调合理的间隔，忘记会进入重学并增加遗忘次数。
- 评分保存前后快照，撤销恢复调度状态且不留下重复有效日志。
- 单词、短语和已掌握错题均能完成“回忆 -> 揭示 -> 评分 -> 排期 -> 到期再现”。
- API、全量测试、数据库完整性和 1440×900 桌面 E2E 通过；本地提交和标签完成，不自动推送 GitHub。

### `v0.8.0-alpha.22.1` 词典查询工作流

**产品目标：** 借鉴欧路、Cambridge、Collins 和 Merriam-Webster 的成熟查询路径，让用户从词形、拼写错误或历史词条快速到达可信结果，并在一个页面完成理解、复制、保存和外部核验。

**内容目标：** 不复制商业词典释义；继续使用现有开放数据和个人语境，商业词典仅提供用户主动打开的合法外链。

**范围：**

- schema 6 本机查询历史、重复查询次数、最近与高频入口。
- 只对完整查询计数，顶部即时联想不写入历史。
- 经本地索引验证的保守词形还原和有边界的拼写建议。
- 词条分区导航、词头与当前释义复制、四个外部词典入口。
- 查询历史保持为兴趣/复习信号，不进入能力分和通用词频。

**非目标：**

- 不抓取牛津、剑桥、柯林斯、Merriam-Webster 或欧路正文。
- 不把拼写建议称为权威纠错，不用激进词干算法猜造词条。
- 不在本版宣称开放中文、例句和词源覆盖已达到商业词典水平。

**验收标准：**

- `inspecting` 能还原并进入 `inspect`，拼错词能显示索引内候选。
- 完整查询持久化、重复计数、快速联想不计数，用户可清空本机记录。
- 左右布局不变，词条可复制、分区跳转、保存并打开外部权威参考。
- 全量测试、语法、差异、schema 6 迁移和桌面浏览器检查通过；本地提交与标签完成，不自动推送 GitHub。

### `v0.8.0-alpha.22` 开放词典分层与可追溯导入

**产品目标：** 把已有大规模 WordNet 从英文语义底座提升为可叠加中文、词源、词形、常用度和真实英汉例句的学习词典，同时让用户清楚每一层是否真实安装。

**非目标：**

- 不抓取或内置牛津、剑桥、柯林斯、欧路等商业词典。
- 不在没有源文件时宣称 Kaikki、Tatoeba 或 wordfreq 已经覆盖真实数据库。
- 不用个人文章出现次数冒充通用词频，不在本版做 FSRS。

**验收标准：**

- 小型合法格式夹具可导入并在同一次查询中返回中文、词源、词形、频率和署名句对。
- 缺作者的 Tatoeba 句对被拒绝，所有开放层保留来源和许可证。
- 旧库幂等迁移到 schema 5，未安装层明确显示“未导入”。

- 不新增考试模板、正式真题内容或 AI 对话助手。
- 不把训练/模考误作题目分类；它们只控制反馈时机。
- 不扩展非 IELTS 的整套试卷能力。

**验收标准：**

- 未完成画像的用户首次进入自动看到画像弹窗，保存后只首次显示快速上手。
- 页面中不再存在通用阅读、选词填空和手动载入题目的重复控件。
- 切换专项题型、训练范围或目标考试后，题目集合与选择一致且旧会话被重置。
- 自动化测试、JavaScript/Python/JSON 语法和差异检查通过；本地提交与标签完成，不自动推送 GitHub。

### `v0.8.0-alpha.13` 双模式工作流与七天分项校准

**产品目标：** 兴趣与备考模式产生明显不同的学习路径，并让训练证据每七天以可解释、低波动的方式校准对应能力分项。

**内容目标：** 同一合法素材可进入沉浸阅读或考试训练，但模式不修改原文；分项训练不被夸大为综合英语或正式考试分数提升。

**范围：**

- 兴趣模式突出兴趣主题、订阅、沉浸阅读、词块和网页素材，隐藏未解决错题压力区。
- 备考模式展示目标、当前最低正确率能力、校准倒计时和弱项题型入口。
- 阅读、听力、词汇、写作和口语分项能力状态与校准历史。
- 每七天在完整训练归档后自动检查，按正确率、题目难度和信心错误微调。
- 单分项至少 8 题、2 个 session、跨 2 天；每周调整幅度有限。
- 综合等级需要至少 3 个有效分项，并同时含输入与输出能力证据。

**非目标：**

- 不把阅读进步外推为听力、口语、写作或综合等级进步。
- 不把 XP、打开文章、看过解析或计划完成度作为能力证据。
- 不输出官方 IELTS/TOEFL 分数预测或心理测量学认证。
- 不在本版加入单题耗时、提示使用、推荐反馈或拉片听力证据。
- 不推送 GitHub，除非用户明确要求。

**验收标准：**

- 两种模式返回不同模式契约、首页信号、主要动作和文章动作。
- 未满七天不运行校准；数据不足的分项保持不变并记录原因。
- 只提供合格阅读证据时仅阅读分项调整，综合等级保持原锚点。
- 校准记录算法版本、周期、证据、调整前后值和综合覆盖情况。
- 训练归档自动检查校准，不要求用户手动触发。
- 自动化测试、语法和差异检查通过；本地提交和标签，不自动推送 GitHub。

- 保存后再次读取的数据与用户设置一致。
- 今日内容返回同一份时长、任务和目标。
- 开启画像推荐时推荐理由说明“对应当前目标”；关闭后显示通用安排。
