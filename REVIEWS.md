# Iteration Reviews

本文件记录每轮大版本的证据、成熟度变化、差异化判断、失败与下一步实验。评分不是宣传数字；没有测试或真实用户证据的能力不会按“完成”计算。

## `v0.8.0-alpha.23.0.3` 考试素材质量门

### 证据与成熟度

- 真实 BBC World 条目只有 19 词且 `content_status=summary`，此前仍可进入题目生成；这是明确的质量门缺失。
- 内容质量由 `6.1/10` 提升到 `6.8/10`：摘要、完整性、污染、重复和考试长度进入同一可解释评分。
- 考试适配由 `6.8/10` 提升到 `7.3/10`：推荐和出题开始使用真实字数门槛，而不只看媒体来源标签。
- 版权与解释性由 `7.0/10` 提升到 `7.6/10`：摘要不再在交互上暗示为原文，受保护全文只通过原站或用户授权导入。
- 趣味性不变：本轮是质量基础设施，不增加玩法。

### 自我反思

过去的 `exam_fit` 主要描述来源与考试主题的匹配，不能证明某篇具体文章适合出题。把 BBC、Economist 或 JSTOR 的品牌当成内容质量代理是不够严谨的。当前门槛解决了最明显的摘要与字数问题，但考试级范围仍偏粗；下一步需要按题型建立长度与篇章结构要求，并对长文生成可追溯的连贯节选，而不是随意截词。

## `v0.8.0-alpha.23.0.2` 练习译文与抓取正文质量

### 证据与成熟度

- 截图与真实活动训练确认问题来自 JSTOR feed 中泄漏的 Gravity Forms JavaScript，而不是普通字符集乱码。
- 训练闭环由 `8.6/10` 提升到 `8.8/10`：练习中心可以从缺译文状态直接完成翻译并回到逐段阅读。
- 内容质量由 `5.5/10` 提升到 `6.1/10`：已有 HTML 节点过滤、明文脚本防线和可审计旧数据修复。
- 可维护性由 `7.3/10` 提升到 `7.5/10`：数据修复进入版本迁移并保留原文，而不是手工改一条数据库记录。
- 内容覆盖不变：本轮提高干净度，不增加来源或可合法使用的全文数量。

### 自我反思

此前把“feed 提供 encoded 内容”近似当成“可读正文”过于乐观，长度阈值只能判断多少，不能判断质量。当前修复针对已有证据且误伤边界较窄，但还没有形成通用正文质量模型；下一步应记录脚本比例、导航噪声、重复率和自然语言句子覆盖，而不是继续添加来源数量。

## `v0.8.0-alpha.23.0.1` 双语义项与双发音修复

### 证据与成熟度

- 代表性 WordNet 查询已具备义项级中文缓存和例句翻译；新增测试覆盖通用 IPA 与带标签 UK/US IPA。
- 学习呈现由 `6.5/10` 提升到 `7.0/10`：中文、英文、例句和个人语境的阅读顺序更直观。
- 查询易用性由 `6.5/10` 提升到 `7.0/10`：不再依赖手动翻译按钮，四类词条均有双发音入口。
- 实际词典覆盖仍为 `5.9/10`：本轮没有新增大型开放数据，许多义项仍缺高质量双语例句。
- 可维护性保持 `7.3/10`：发音选择与缺失翻译判断集中为可测试函数，schema 不变。

### 自我反思

这次修复的是信息组织和已有数据的利用率，不是词典规模。双语音按钮可以使用不同系统语音，但若来源只有通用 IPA，就不能把相同音标包装成英美差异。下一步真正提高词典分数仍依赖合法开放数据导入、例句质量筛选和多义词抽检。

## `v0.8.0-alpha.23.0` 统一记忆复习闭环

### 本轮假设

如果单词、短语和已掌握错题进入同一到期队列，并要求用户先主动回忆再评分，原有阅读、查词和错题素材就能真正形成跨天记忆闭环，而不是停留在收藏数量。

### 已获得证据

- schema 7 对旧生词和已掌握错题幂等回填，新素材自动注册且不重复。
- 调度测试证明四档初始间隔单调递增，遗忘进入重学并累计 lapse。
- 评分保存前后完整快照；十分钟内撤销可恢复每个调度字段并回退同日计划进度。
- 未到期项目拒绝重复评分，避免双击或网络重试连续推进间隔。
- 隔离数据库桌面 E2E 验证固定侧栏、左右队列、揭示、四档评分、排期和撤销，不污染真实生词本。
- 135 项自动化测试、Python/JavaScript 语法和差异检查通过。

### 成熟度变化

| 维度 | 上版 | 本版 | 判断 |
| --- | ---: | ---: | --- |
| 复习调度 | 3.0/10 | 5.8/10 | 已形成正式可执行循环，但算法仍是版本化基线而非 FSRS |
| 学习闭环 | 8.3/10 | 8.6/10 | 阅读/查词/错题终于进入跨天到期复习 |
| 可维护性 | 7.0/10 | 7.3/10 | 调度独立成领域模块，E2E 使用隔离数据库 |
| 解释与控制 | 6.4/10 | 7.0/10 | 用户看到预测间隔，可筛选、撤销且不静默改能力分 |
| 趣味性 | 4.8/10 | 5.0/10 | 完成感增强，但仍没有字幕、挑战与长期留存证据 |

### 自我反思

这一版解决了“收藏以后怎么办”，但尚未证明排期最优。`adaptive-interval-v1` 只是一套保守、确定、可测试的迁移基线；将它包装成 FSRS 会损害项目可信度。当前卡片答案主要依赖保存原句，中文多义项和高质量搭配仍受词典数据覆盖限制。

### 下一步实验

1. 获得许可明确、版本锁定的 FSRS 实现并用官方/上游测试向量核验。
2. 增加目标保留率、每日新卡上限、暂停和重置，但默认值保持简单。
3. 记录至少七天真实评分日志，比较预测到期与实际遗忘，不用 XP 代替记忆效果。
4. 后续把字幕句子接入同一复习接口，而不是再建一套平行收藏系统。

## `v0.8.0-alpha.22.1` 词典查询工作流

### 本轮假设

在不复制商业词典内容的前提下，补齐词形还原、拼写建议、查询历史和页内导航，可以减少用户从“我大概记得这个词”到“看到可学习义项”的操作成本。

### 已获得证据

- 词形候选只有在精选词条、WordNet 或开放词条索引真实存在时才接受。
- `inspecting` 可回到 `inspect` 并复用 WordNet 义项；`inspeckt` 可得到索引内拼写候选。
- API 测试证明快速联想不落库、完整查询重复计数并可清空。
- 页面保留固定侧栏和左右主从结构，增加分区导航、复制与四个外部核验入口。
- 126 项自动化测试、语法和差异检查通过；真实数据库备份后迁移到 schema 6 且完整性为 `ok`。
- 仓库内桌面 E2E 在 1440×900 验证词形、拼写、历史、固定侧栏、左右布局、分区和外链并生成截图。

### 成熟度变化

| 维度 | 上版 | 本版 | 判断 |
| --- | ---: | ---: | --- |
| 查询易用性 | 5.2/10 | 6.5/10 | 从精确输入依赖提升为可恢复、可回看路径 |
| 实际词典覆盖 | 5.8/10 | 5.9/10 | 只改善到达方式，真实数据仍未补齐 |
| 隐私与解释性 | 5.5/10 | 6.4/10 | 本机历史可清空，且不冒充能力证据 |
| 差异化 | 6.5/10 | 6.7/10 | 历史可继续连接个人语境和词组本，但尚未接复习调度 |

### 自我反思

成熟查询动作并不是产品壁垒，它们只是词典达到可用门槛所需的基本功。真正有差异的仍是查询结果能回到用户读过的原句、保存整个词组，并在未来进入同一复习队列。本版没有真实数据审计，也没有证明建议能减少查词时间，因此词典整体仍不能评为 6 分。

### 下一步实验

1. 导入许可明确的真实高频开放数据并完成代表性抽检。
2. 记录“建议被点击、结果被保存、回到语境”的漏斗，不记录每次键入。
3. 把重复查询与词卡复习优先级连接，但用后续回忆结果验证，而不是直接加能力分。

## `v0.8.0-alpha.22` 开放词典分层

### 本轮假设

如果词典把语义、词源词形、双语例句、通用频率和个人语料明确分层，并能说明每层是否真实安装，用户会更容易先理解常用意义，再按需追根溯源，同时项目可以合法替换和扩充数据。

### 已获得证据

- 真实本机数据库已有 107,519 个 WordNet synset、185,129 条 lemma 和 132,238 条关系。
- 小型格式夹具成功合并 Kaikki 中文/词形/词源、wordfreq Zipf 频率与 Tatoeba 署名英汉句对。
- 缺少任一作者的 Tatoeba 句对被拒绝。
- 英文查询和中文反查均能返回开放词条；来源、许可证、通用频率和个人频率保持分离。
- schema 5、FTS、流式导入与页面契约均有自动化覆盖。

### 成熟度变化

| 维度 | 上版 | 本版 | 判断 |
| --- | ---: | ---: | --- |
| 词典工程基础 | 4.8/10 | 6.4/10 | 从 WordNet 专用查询提升为可替换的多源分层系统 |
| 实际词典覆盖 | 4.5/10 | 5.8/10 | WordNet 覆盖强，但真实 Kaikki/Tatoeba/wordfreq 尚未导入 |
| 来源可信度 | 5.5/10 | 7.0/10 | 来源、许可证、作者和安装状态可见 |
| 学习呈现 | 5.5/10 | 6.5/10 | 中文确认、英文概念、常用度、词源和两类语境分层 |
| 工程可维护性 | 6.7/10 | 7.0/10 | 新增独立 lexical_data 模块，导入逻辑不再塞入主服务 |

### 差异化与自我反思

本轮借鉴了成熟词典的多义项、词源、常用度和例句组织，但没有复制商业词典内容。真正有价值的是开放词典与用户自己文章、阅读器、词组本和训练证据处于同一闭环。

最大的限制是数据工程与内容覆盖不能混为一谈。夹具证明导入和查询正确，不证明数十万条 Wiktionary 翻译都自然，也不证明 Tatoeba 每个句对都适合学习。因此实际词典评分只升到 5.8，而不是为了满足目标直接写成 6 分。

### 下一步实验

1. 用户提供或下载 Kaikki、Tatoeba 与 wordfreq 合法源文件后，先导入 B1-C1 高频精选集。
2. 抽检至少 100 个多义词、50 个短语、50 个中文反查和 50 个低频词。
3. 记录错误翻译、过时词义、例句不自然和词性错配，建立可回滚黑名单。
4. 通过质量门槛后将实际词典覆盖提升到 6.3，再进入 FSRS 复习调度。

## `v0.8.0-alpha.21` 文章池多布局与素材边界

### 本轮假设

保留高效率左右精读，同时提供信息更丰富的网格发现和明确素材归属，应能降低“文章多了以后找不到”的成本，又不打断既有学习闭环。

### 已获得证据

- 118 项自动化测试通过，覆盖布局契约、偏好保存、公开/私人筛选、计数和迁移幂等。
- Python 编译、JavaScript 语法和差异检查通过。
- 数据库迁移测试发现并修复了极简旧表缺少 `source_url` 时的兼容错误。
- 固定侧栏与默认左右主从 CSS 契约保留，网格卡片复用现有阅读、出题和今日计划动作。

### 成熟度变化

| 维度 | 上版 | 本版 | 判断 |
| --- | ---: | ---: | --- |
| 内容发现 | 5.0/10 | 6.2/10 | 从单一列表提升为两种工作视图并提供真实素材范围 |
| 学习闭环 | 8.2/10 | 8.3/10 | 发现卡片直接进入既有阅读、训练和计划链路 |
| 数据边界 | 4.5/10 | 5.5/10 | 私人材料有显式字段和迁移，但尚无账号权限隔离 |
| 工程可维护性 | 6.5/10 | 6.7/10 | 新迁移和设置接口可测试，前端渲染函数仍偏大 |
| 公共网站准备度 | 3.8/10 | 4.0/10 | 数据归属更清楚，仍缺认证、云存储和安全授权 |

### 差异化与自我反思

网格本身是成熟内容产品的常规能力，不构成壁垒。真正有价值的是每张卡片能立即进入“一文多用”的阅读、考试训练和计划闭环，同时不把私人 EPUB 或网页摘录混入公共素材。

本轮仍没有真实用户数据证明网格能缩短选文时间，也没有仓库内 Playwright E2E。卡片里的“学习状态”目前只是完整正文、摘要或今日推荐，不是可靠的未读、在读、已完成状态。筛选条件与合辑也尚未实现，不能把文章管理称为成熟资料库。

### 下一步实验

1. 记录布局切换、选文耗时、卡片到训练的转化，不用主观偏好代替效率证据。
2. 建立未读、在读、已完成和稍后读状态，并与每日计划去重。
3. 保存筛选条件，增加用户合辑与收藏，但避免把侧栏变成媒体名称清单。
4. 把文章查询、筛选和权限逻辑从主服务抽到内容领域模块。
5. 回到训练主线，记录处方采纳与下一组表现，再推进 FSRS 和字幕拉片。

## `v0.8.0-alpha.20` 训练状态与个性化处方

### 本轮假设

如果训练状态由服务端保存，并把正确率、用时、改答、提示和信心组合成可解释处方，用户重新打开应用时应更容易继续，也更清楚下一步为什么练这一类题。

### 已获得证据

- API 测试覆盖活动训练创建、更新、跨客户端读取、完成、历史关联和完成后退出活动状态。
- 处方测试验证正确率、独立题目数、平均用时、改答、提示和确定但答错来自数据库记录。
- 116 项自动化测试通过，固定侧边栏和左右布局契约未回退。
- 数据库迁移版本 3 可从旧库幂等升级。

### 成熟度变化

| 维度 | 上版 | 本版 | 判断 |
| --- | ---: | ---: | --- |
| 本地训练闭环 | 7.5/10 | 8.2/10 | 从本地草稿提升为服务端活动状态，并关联正式历史 |
| 自适应建议 | 4.5/10 | 6.2/10 | 已有多证据处方，但仍是启发式规则 |
| 数据连续性 | 5.5/10 | 7.2/10 | 可跨刷新和浏览器恢复；仍无账号与跨设备云同步 |
| 工程可维护性 | 6.0/10 | 6.5/10 | 新增独立 `practice_state` 模块；主服务仍偏大 |
| 公共网站准备度 | 3.5/10 | 3.8/10 | 状态模型更接近多用户，但仍固定为本地学习者 |

### 差异化反思

增强的不是“也有一个推荐算法”，而是处方可以回到具体行为证据，并直接进入同题型训练。它和原文证据、错因、同类题、掌握状态处于同一条链路，这比孤立的 AI 推荐更有产品价值。

但这还不是壁垒。成熟产品也能统计正确率和时间；只有当处方能持续提高下一组表现，并且推荐理由比普通错题统计更准确时，差异化才成立。目前尚无真实用户对照数据证明这一点。

### 本轮发现的问题

1. 初次外键修改曾误命中前一个表，说明大文件内机械编辑风险仍高；建库测试及时阻止了发布。
2. `alpha.19` 虽保存了单题行为字段，训练详情查询最初没有取出这些字段；本轮补齐。说明“写入成功”不等于“闭环可见”。
3. 仅按作答次数会让反复练同一题虚增证据，本轮增加独立题目覆盖和处方置信度。
4. 活动训练反馈快照会复制部分解析数据，长期可能使数据库膨胀，需要后续改为引用尝试记录或压缩状态。
5. 75 秒/45 秒参考值是初始启发式，不是官方标准，也没有按考试、题型和用户水平校准。

### 尚未证明

- 用户是否愿意采纳处方，以及采纳后下一组正确率是否提高。
- 处方的 5/10 题数量是否适合不同可用时间。
- 多浏览器同时编辑同一活动训练时的冲突策略。
- 无 Playwright 的情况下，真实浏览器恢复路径和无障碍体验仍缺少自动验收。

### 下一步实验

1. 记录处方展示、采纳、修改、忽略和训练后表现，形成推荐反馈闭环。
2. 比较“按最低正确率推荐”和“多证据优先级推荐”的下一组提升幅度。
3. 增加活动状态修订号，检测多客户端覆盖冲突。
4. 建立画像到训练、刷新恢复、完成归档、处方再练的核心浏览器 E2E。
5. 完成文章池多布局后，继续 FSRS 复习和字幕拉片，而不是重新扩张内容来源。
