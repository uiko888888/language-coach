# Product Decisions

本文件记录会影响多个版本的产品决定。新决定追加记录，不静默改写历史。

## 2026-07-18 - 正文提取先可审计与收集标签，再训练分类器

**决定：** 原始抓取先写入审计记录；正文通过来源适配器和语义区块规则提取，并记录版本与置信度。用户纠错形成标签，达到可评估规模后才训练小型区块分类器；模型结果仍必须通过确定性质量门。

**原因：** 当前错误样本数量不足，直接微调会把来源偶然结构和错误标签学进模型，也难以解释误删。来源规则对已知媒体更稳定，反馈数据则能决定模型是否真的优于规则。

**约束：** 模型不得改写作者原文，不得绕过版权状态，不得把图片说明或披露送入出题器；每次提取必须能追溯原文、版本和用户反馈。

## 2026-07-18 - 复习数据模型与调度算法解耦

**决定：** 单词、短语和已掌握错题共用 `review_items` 与 `review_logs`，调度算法通过独立模块和 `scheduler` 版本字段替换。当前实现明确标记 `adaptive-interval-v1`，不称为 FSRS。

**原因：** 当前网络环境无法安装并核验 `py-fsrs`。如果为了进度复制不完整公式，会制造不可审计的记忆排期；先稳定项目、日志、撤销和 API，再以测试向量迁移算法，风险更低。

**约束：** 每次评分必须保存前后快照；未到期项目不可重复评分；撤销窗口为十分钟；评分只增加复习进度，不直接改变 CEFR 或考试能力。

## 2026-07-18 - 查词行为是复习兴趣信号，不是能力证据

**决定：** 只记录用户明确提交或打开的完整查询；输入框即时联想不写入历史。重复查询可用于生词优先级和后续复习提醒，但不得直接改变 CEFR、考试能力或通用词频。

**原因：** 重复查词可能表示陌生、感兴趣、核对拼写或只是操作重复，证据含义不足以推断语言能力。用户可以清空本机记录，未来云端版需要独立隐私授权。

**约束：** 商业词典只提供外链或正式授权 API；拼写与词形建议必须标注为候选并由本地索引验证。

## 2026-07-18 - 词典按数据职责分层，安装状态必须可见

**决定：** WordNet 只负责英文语义与关系；Kaikki/Wiktionary 负责词形、词源和开放词条；Tatoeba 负责可署名英汉句对；wordfreq 负责通用频率；个人文章只负责本机语境和个人出现次数。

**原因：** 成熟词典不是把不同质量的数据混成一个答案。分层才能解释来源、替换错误数据，也能避免机器翻译、个人语料和出版词典释义相互冒充。

**约束：** 页面必须显示来源与许可证；未导入的数据层显示“未导入”；缺作者的 Tatoeba 句对拒绝入库；个人出现次数不得标为通用高频；商业词典必须通过正式授权 API 或外链。

## 2026-07-18 - 左右精读是默认布局，网格只承担发现

**决定：** 文章池默认继续使用标题列表在左、内容详情在右的主从布局；网格是可选发现视图。布局与密度保存在用户设置中，手动、插件和私人 EPUB 素材拥有明确的 `private` 归属。

**原因：** 左右布局适合连续筛选和精读，网格适合先比较题材与难度。两者服务不同阶段，不应互相替代。显式素材归属也是未来云端多用户权限的必要前置条件。

**约束：** 全局固定左侧栏不能因文章布局改变；私人材料不得进入公共内容池；卡片动作必须复用阅读、训练和今日计划闭环。正式云端上线前仍需账号所有权和后端权限校验，`visibility` 字段本身不是安全边界。

## 2026-07-18 - 活动训练以服务端状态为准，处方必须解释证据并可拒绝

**决定：** 浏览器本地草稿保留为兼容回退，正式活动训练由服务端记录；训练处方同时报告行为指标、独立题目覆盖和理由，用户可以选择题量、关闭推荐或放弃。

**原因：** 本地草稿无法支持跨浏览器连续性，也不能成为未来账号同步基础。只按错误率推荐会忽略速度、犹豫和误判信心，但黑箱权重又会削弱用户控制。

**约束：** 处方不得称为分数预测；重复做同一题不能伪装成广泛证据；兴趣模式不强推考试处方；活动状态完成后必须关联正式训练历史。多用户上线前必须把固定本地学习者替换为账号所有权和并发修订控制。

## 2026-07-18 - 暂停内容扩张，先完成可验证训练闭环

**决定：** `alpha.19` 起暂停增加媒体、RSS 和大型词典数据，优先完善答题证据、错题教学、同类验证、掌握状态和下一轮训练。

**原因：** 当前素材已足够验证核心学习流程，继续增加来源不会修复训练教学深度、状态连续性和个性化不足。成熟产品的壁垒来自学习结果闭环，而不是入口数量。

**约束：** 新功能必须说明如何缩短“训练到有效反馈”的路径；内容需求保留在待办，但不挤占训练闭环版本。规则生成的解析必须明确质量边界，不能冒充官方或人工专家解析。

## 2026-07-18 - 采用行为保持的渐进模块化，不做一次性重写

**决定：** 先抽版本、迁移和备份等稳定基础层，再按内容、练习、词典、画像拆后端；前端先抽 API 和状态，再逐页迁移。`server.py` 在过渡期继续作为组合根。

**原因：** 当前单文件已经偏大，但已有大量可工作的业务闭环和集成测试。一次性改写会扩大回归面，也无法直接提升用户价值；明确依赖方向并逐模块抽离更容易核验。

**约束：** 新数据库变化只能进入迁移注册表；底层模块不得反向依赖 HTTP 主模块；每次抽离必须保持公开行为并通过完整测试。前后端版本不一致必须显式提示。

## 2026-07-18 - 来源注册、内容获取和全文使用是三个独立权限层

**决定：** 来源可以先登记在内容注册表中，但只有接入方式和版权状态允许时才自动获取；获得标题或摘要也不等于获得全文训练与再分发权限。

**原因：** 新闻 RSS、付费媒体、流媒体、学校邮件、公共领域图书和用户自有文件具有完全不同的权限边界。统一按“能在网上看到”处理会制造版权和产品可信度问题。

**约束：** 注册表必须记录接入方式与版权处理；自动来源采用 RSS/API，付费来源默认元数据与外链，浏览器摘录必须由用户主动授权，本地字幕和私人材料不进入公共内容库。

## 2026-07-18 - 中文释义必须标明数据层级，不用机器翻译冒充出版词典

**决定：** 英文语义关系以 Open English WordNet 为底座；中文依次采用许可明确的开放双语词典、人工精选词条和可追溯的机器翻译缓存。商业词典只通过正式授权 API 或外链使用。

**原因：** “翻译得出来”和“词典义项可靠”不是一回事。机器翻译适合补齐当前语境，但不能替代经过编辑的多义项、语域和搭配说明；来源不透明也会让错误难以纠正。

**约束：** 页面必须显示英文语义、中文翻译和搭配的来源；个人语料频次不能称为通用高频；Kaikki/Tatoeba 导入必须保留许可证、署名和单条来源元数据。

## 2026-07-18 - 学习画像归用户所有，自动校准只更新能力证据

**决定：** 用户中心是目标、兴趣、内容偏好和自报基线的长期管理入口。用户可以随时修改这些设置；系统每七天只根据达到门槛的训练证据更新对应能力分项与推荐，不覆盖用户目标和偏好。

**原因：** 个性化需要持续进化，但若系统静默改写用户目标，会让推荐不可理解也不可控制。把主动设置与测量证据分开，既保留私人教练感，也保留用户最终决定权。

**约束：** XP、计划完成度和页面浏览不作为能力证据；每日计划只维护一套可编辑数据；未来 AI 建议也必须允许用户拒绝或修正。

## 2026-07-18 - 用户只看到考试题型，通用生成模式留在引擎内部

**决定：** 练习中心的主层级固定为目标考试、反馈方式、训练范围和考试专属题型。阅读、选词等通用生成模式不再作为额外的用户下拉选项；单篇组合由当前考试的题型模板混合生成。

**原因：** 同时展示通用模式和考试题型会形成重复分类，而且用户切换标签时难以判断题目是否真的变化。产品界面应使用用户熟悉的考试语言，内部引擎再负责复用生成能力。

**约束：** 训练/模考只决定答案反馈时机；专项、单篇和整套决定训练范围；每次考试、范围或题型切换必须清空不匹配的旧会话并加载或生成对应题目。

## 2026-07-17 - 描述性趋势不冒充能力分

**决定：** 第一版跨 session 趋势展示累计正确率、最近五题与此前五题差值，并明确标为表现趋势；不把它转换为 CEFR、IELTS 或 TOEFL 预测分数。

**原因：** 少量、难度不均的规则模拟题不足以支撑稳定能力估计。先让历史可追溯，再逐步加入题目难度、时间、置信度和知识追踪，比直接给出漂亮但不可信的分数更稳妥。

**约束：** XP 不进入正确率和能力趋势；未作答保留在 session 分母；后续预测必须报告样本量、时间范围和置信边界。

## 2026-07-17 - TOEFL 阅读先独立题型，再扩完整科目

**决定：** TOEFL 首先独立实现事实、推断、主旨、句子简化和语境词义，并复用现有证据、会话、错因和复盘基础；插入、总结、完整阅读套题及听说写后续分版完成。

**原因：** 入口和题型名称不能代替考试拟合。先消除五类现有入口的通用生成痕迹，能建立可验证的质量底座，同时避免一次铺开 TOEFL 四科导致题目质量失控。

**约束：** 当前题目必须标记为规则模拟，不声称为 ETS 真题；官方材料只保存链接或接受用户合法私有导入。

## 2026-07-17 - 学习计划由用户控制，推荐保持可解释

**决定：** 系统按可用时间、学习内容组合、近期/长期目标、模式和订阅生成今日安排，但用户可以修改任务顺序、关闭画像推荐或直接进入其他模块。

**原因：** 私人教练需要减少“今天学什么”的决策成本，但不能覆盖用户已有的学习计划。推荐只有在用户理解原因并能纠正时才会改善画像。

**约束：** 目标和兴趣不直接增加能力分；关闭画像推荐后退回通用安排。系统级通知、浏览器通知和跨设备推送必须独立授权，不能与应用内今日内容混为一谈。

## 2026-07-17 - GitHub 推送改为显式动作

**决定：** 每轮迭代完成后执行本地测试、提交和标签；只有用户明确要求时才推送 GitHub。

**原因：** 本地版本管理和远程发布是两个不同动作，用户希望先体验并确认每轮成果，再决定是否发布。

**约束：** 未收到明确推送指令时，不执行 `git push`；提交和标签仍必须使用规范的 Conventional Commit 与版本号。

## 2026-07-17 - 信心校准用于区分知识与猜测

**决定：** 练习与模考记录“猜测、犹豫、确定”三级作答信心，并在结果中优先暴露“确定但答错”。

**原因：** 只看正确率会把蒙对误判为掌握，也会隐藏稳定但错误的理解。信心与正误组合比单一积分更适合决定下一轮训练。

**约束：** 信心是用户自报数据，不直接等同于能力分；后续只能与题目难度、历史表现和复习结果共同使用。

## 2026-07-17 - 插件采集必须回到学习闭环

**决定：** 浏览器扩展保存的单词、段落和正文进入主应用学习队列，并保留来源网页、文章、查词和出题入口。

**原因：** 插件若只负责收藏会形成新的信息堆积；采集内容只有能继续阅读、查词、训练和复盘，才会转化为学习资产。

**约束：** 主应用复用既有令牌桥接和后端数据，不另建不一致的插件数据库；原始网页版权与来源地址必须保留。

## 2026-07-17 - 考研先完成阅读专项再扩完整试卷

**决定：** 考研英语先独立实现细节推断、主旨态度、长难句语义和完形逻辑，再拆分英语一/二、新题型、翻译和写作。

**原因：** 当前用户的首要目标是快速提升阅读；同时铺开作文评分和完整套卷会稀释题型质量。

**约束：** 当前考研题必须标记为规则模拟；没有英语一/二套卷引擎前，界面不得开放整套按钮。

## 2026-07-17 - 题目来源必须可追溯

**决定：** 题库区分官方公开样题、授权材料、用户自有/获授权导入和系统模拟题；每条资源记录提供方、年份、来源链接、访问方式和版权状态。

**原因：** “网上能搜到”不等于可以复制、再分发或当作真题使用。来源标签是题目可信度和版权边界的一部分。

**约束：** 未验证授权的历年真题只提供外部链接或允许用户私有登记，不抓取全文、不提交仓库；系统生成内容必须标记为模拟题。

## 2026-07-17 - 训练反馈与模考反馈分离

**决定：** 练习中心提供训练模式和模考模式。训练模式逐题反馈；模考模式只保存答案和标记，统一交卷后才展示答案、证据与错因。

**原因：** 即时讲解适合学习新方法，延迟反馈适合检验真实表现；把两种目的混在一个交互里会泄露答案并扭曲成绩。

**约束：** 两种模式共享题目校验、评分和错因规则；未作答必须进入完成度和模考分母，XP 仍与能力表现分开。

## 2026-07-16 - 双模式共享同一画像

**决定：** 产品提供兴趣模式和备考模式，但共享词汇、行为、能力和复习数据。

**原因：** 用户希望自由选择当天学习内容，同时让兴趣输入帮助真实能力提升。

**约束：** 兴趣行为可以校准画像，但不能直接虚增 IELTS/TOEFL 预测分数。

## 2026-07-16 - 首次画像分为两条路径

**决定：** 已有测试成绩的用户填写成绩与目标；暂无成绩的用户可自评、参加快速测试或先跳过。

**原因：** 部分用户已有 EF SET、IELTS 或 TOEFL 分数，重复强制测试会增加进入成本。

## 2026-07-16 - 画像持续进化

**决定：** 画像不是一次性问卷。有效训练后微调，每周总结，每月建议局部复测，并允许用户纠正。

**原因：** 语言能力、目标和兴趣会变化，静态标签无法长期指导推荐。

## 2026-07-16 - 能力分与 XP 分离

**决定：** XP 表示投入和完成度，能力分表示可验证表现。

**原因：** 防止通过重复简单任务获得不可信的考试能力增长。

## 2026-07-16 - 兴趣内容作为可订阅大板块

**决定：** 首批内容覆盖字幕、人物访谈、新闻、小说和博客，用户可以订阅并接收应用内更新。

**原因：** 兴趣内容需要成为长期输入来源，而不是一次性的文章筛选条件。

## 2026-07-16 - 一份素材多次利用

**决定：** 所有合规素材使用统一流水线，提取生词、词块和重点句，并转化为阅读、听力、填空、词汇和复述练习。

**原因：** 降低寻找素材的成本，让每次真实输入都能进入练习、复盘和画像闭环。

**约束：** AI 生成内容标记为模拟题；原文、观点、翻译和生成内容必须可区分。

## 2026-07-16 - IELTS 优先建立完整纵向闭环

**决定：** `v0.8.0` 优先完整实现 IELTS 阅读训练，底层预留 TOEFL；`v0.9.0` 再完整展开 TOEFL 和 AI 听力。

**原因：** 同时铺开多个考试会让题型模板和解析质量失控。

## 2026-07-16 - 插件归属现有版本历史

**决定：** 浏览器插件属于已经实现的 `v0.7.0-alpha.5/.6`，不再按照旧计划归入未来 `v0.9.0`。

**原因：** 路线图必须反映真实 Git 历史，不能追溯改写已发布版本。

## 2026-07-16 - 成绩选择填写但至少一项

**决定：** IELTS、TOEFL、EF SET 或其他测试的总分和分项成绩允许选择填写，但至少提供一个有效成绩。

**原因：** 兼容只有总分、只有部分分项或来自不同测试体系的用户，同时避免建立完全没有依据的“已有成绩”画像。

## 2026-07-16 - 订阅首先进入今日内容

**决定：** 第一阶段的订阅更新集中在应用内“今日内容”。浏览器徽标和通知后续实现，且由用户主动开启。

**原因：** 先验证推荐价值和打开率，避免用高频通知打扰用户。

## 2026-07-16 - 推荐组合与自由选择并存

**决定：** 打开素材时展示系统推荐的学习组合，同时允许用户单独选择阅读、听力、词汇、填空或复述。

**原因：** 推荐提高效率，自由选择保留兴趣模式的自主感。

## 2026-07-16 - 流媒体按形态和时长组织

**决定：** 流媒体区分经典剧集、传统综艺/访谈、流媒体综艺和短视频，并按微片段、短片段、完整单元和长内容筛选。

**原因：** 内容类型不能代表学习成本，用户需要根据可用时间选择素材。

## 2026-07-16 - 考试拟合不是复制真题

**决定：** 使用考试题型结构、能力标签和难度规则生成模拟题，不复制官方或商业题目文本。

**原因：** 保持训练价值和版权边界，同时避免换词式仿题产生不自然、偏怪或答案模糊的问题。

**约束：** 题目必须通过答案唯一性、原文证据、自然度、干扰项和目标难度校验；不合格时拒绝展示。

## 2026-07-16 - 扩展国内考试目录

**决定：** 在 IELTS、TOEFL 等国际考试之外，将四级、六级和考研英语作为正式考试标识接入统一训练体系。

**原因：** 内容学习和能力标签可以共享，考试入口需要覆盖更广泛的真实提升目标。

**约束：** 每种考试必须有独立题型名称、提示语、干扰项逻辑和讲解标签。基础风格生成器不应被描述为完整官方题型或完整套卷。

## 2026-07-16 - 来源注册表区分接入与版权模式

**决定：** RSS、付费媒体、开放全文、学术索引和流媒体使用不同接入模式，不把所有来源都当作可抓取全文。

**原因：** 来源价值、技术能力和版权许可是三个不同问题，必须在进入内容池前明确。

## 2026-07-16 - 事件聚类延后

**决定：** `v0.7.0-alpha.8` 先交付来源订阅和今日内容，持续事件聚类延后到来源去重与内容量稳定后。

**原因：** 当前仅凭关键词聚类容易把不同事件合并，或把同一事件拆散，展示错误时间线比暂不展示更糟。

## 2026-07-16 - 译文逐段跟随原文

**决定：** 阅读台、文章详情和做题原文统一按段落显示原文与对应译文。一键翻译和手动译文都使用空行保持段落对齐。

**原因：** 逐段对照减少视线跳转，也能保留原文作为主要学习对象。

## 2026-07-16 - 陌生词候选不冒充完整画像

**决定：** 动态画像完成前，使用 B1 默认等级、词典等级、词长启发式和生词本记录初筛“可能不认识”的词与短语。

**原因：** 先改善文章分析的实际相关性，同时明确当前判断的置信度边界。

## 2026-07-16 - 双模式先落地推荐策略，再接动态画像

**决定：** `v0.7.0-alpha.10` 先提供可持久化的兴趣/备考模式和两套今日推荐权重；画像、目标日期、弱项和能力预测在 `v0.8.0` 逐步接入。

**原因：** 用户现在需要一个直观入口决定今天怎么学，但当前数据不足以声称推荐已经个性化到能力层面。

## 2026-07-16 - 翻译密钥只进入本地私密配置

**决定：** DeepL 和 LibreTranslate 配置只从环境变量或被 Git 忽略的 `.env.local` 读取。应用不把密钥写入数据库、前端或仓库。

**原因：** 本地 MVP 需要易配置，同时必须避免凭据泄漏。没有提供商时保留手动逐段译文，不调用来源不明的公共代理。

## 2026-07-17 - 未收录不等于不可查询

**决定：** 任意合法英文单词或最多六词短语都返回可操作的查询对象。完整开放词典尚未导入时，仍展示个人语境、已有翻译、生词状态、发音、DeepL 动作和欧路外链，并明确标注释义未收录。

**原因：** 查询、保存和回到语境必须形成连续流程；但用自动翻译冒充完整词典会隐藏词义、语域和搭配差异。

## 2026-07-17 - 生词重复保存更新原记录

**决定：** 同一词或短语再次保存时更新已有卡片的最新有效语境，不创建重复卡片，也不自动重置用户已经形成的熟悉状态。

**原因：** 重复查询本身是学习信号，应该丰富同一学习对象，而不是污染生词本。

## 2026-07-17 - WordNet 中文采用逐义项翻译缓存

**决定：** 保留 Open English WordNet 的英文原文，按义项和例句分段调用已验证的翻译服务，并把中文逐条对齐显示、按文本哈希缓存。配置存在不等于服务可用，DeepL 必须经过运行时验证后才自动翻译。

**原因：** WordNet 本身不含中文。只翻译词头会丢失具体词义，伪装成双语词典也不可靠；逐条翻译能保留英文思维入口，并在服务不可用时继续使用已缓存译文。

## 2026-07-17 - IELTS 新题必须先校验再入库

**决定：** IELTS 判断题、标题匹配、信息匹配和摘要填空使用独立生成器。每道题保存题型、能力、难度、生成来源和校验报告；证据、答案、选项与题型约束未通过时拒绝入库。

**原因：** 只替换考试标签无法产生可靠专项训练。校验契约也是后续 AI 生成、人工评价和模型微调能够共享的质量底座。

**约束：** 当前规则模板标记为模拟题，不声称复刻官方真题；历史通用题保留数据但不混入当前 IELTS 专项训练。

## 2026-07-17 - 今日进度只记录可验证动作

**决定：** 做题在训练归档时计数，错题只在首次解决时计数，词汇只在新建词卡时计数；阅读由用户手动确认，加入计划或打开文章本身不算完成。

**原因：** 今日计划需要减少用户操作，但自动计数不能把浏览、重复保存或重复点击误报成学习成果。

**约束：** 队列加入与完成必须幂等；计划百分比和 XP、能力趋势彼此独立，不用完成计划虚增考试能力。

## 2026-07-17 - TOEFL 总结题分阶段实现

**决定：** `v0.8.0-alpha.11` 先实现有跨段覆盖校验的四选一最佳总结，明确标注为“单选基础”；正式六选三总结待多选答题、部分得分和套题计分模型完成后再启用。

**原因：** 当前训练组件只支持单选。直接把答案拼成字符串会让交互、评分和错因失真，也会把基础模拟题误呈现成 ETS 正式题型。

**约束：** 页面和题目注释不得把单选基础称为正式六选三；后续多选实现必须验证三个主旨覆盖、干扰细节和无序答案评分。

## 2026-07-17 - 新版考试题型先模拟、后复刻

**决定：** `Complete the Words` 首版只交付可验证的单空训练单元，并用 `official_equivalence=false` 明确它不是 ETS 官方界面复刻。

**原因：** 当前环境无法访问 ETS 页面，且现有题目模型是一题一答案。未经官方样题核验就固定多空数量、字母比例和评分，会制造虚假的“正规题型”。

**约束：** 每题必须保存目标词、前缀、缺失字母数、掩码文本和生成来源；官方规格核验后才能增加多空任务和考试计时。错词可进入个人生词本，但不得把 ETS 或商业题目正文打包分发。

## 2026-07-17 - 私人 EPUB 与可分发内容分离

**决定：** EPUB 作为用户本机私人素材进入独立书库，只有用户选中的章节才转换为文章；原文件、完整路径和正文不进入 Git 或远端服务。

**原因：** 长篇书籍需要章节化学习，但本地可读不等于项目有权重新分发。独立书库也能避免数百章直接淹没文章池。

**约束：** 所有书籍标记 `private_user_material`；列表接口隐藏完整路径和正文；导入器不提供下载、分享或版权判断，文件安全检查必须先于解析。

## 2026-07-18 - 本地内容更新采用机会式调度

**决定：** 后端启动时检查内容新鲜度，运行期间每六小时调度；保留手动刷新。单来源失败独立记录并指数退避，用户手动刷新可绕过退避。

**原因：** 当前产品是本地单用户应用，后端退出或电脑关机时无法可靠执行常驻任务。机会式调度能先保证打开应用时内容更新，而不伪装成云端全天候服务。

**约束：** RSS 摘要不能标记成全文；推荐时效使用发布时间；网络请求不持有数据库事务；URL、GUID 和正文哈希按顺序去重；系统级后台任务和推送必须由用户明确开启。

## 2026-07-18 - 公共产品采用 Web/PWA 加可选本地伴侣

**决定：** 面向所有用户的主产品采用托管 Web/PWA，公共文章由云端计划任务统一刷新；Windows 登录任务保留为当前本地版和未来本地伴侣的启动方式。EPUB、私有字幕、本地词典与用户私密密钥默认留在本机。

**原因：** 纯桌面软件会把更新、安装和跨设备同步成本转嫁给每个用户；纯网站又难以安全处理私人文件和本地插件闭环。混合架构能让公共内容可靠更新，同时保留本地隐私能力。

**约束：** 当前 Windows 任务只在用户登录且电脑清醒时工作，不等同于云端全天候更新。公开注册前必须补账号权限、云数据库、持久任务队列、监控、备份、限流和数据删除。

## 2026-07-18 - 首次画像允许多入口且不等同于考试预测

**决定：** 用户可通过已有成绩、阅读词汇快速基线或 CEFR 自评建立初始画像。画像证据按正式成绩、快速基线和自评区分置信度；成绩日期越久置信度越低。画像能力、XP 和计划完成度分别记录。

**原因：** 通用产品不能假设每个用户都有 IELTS/TOEFL 成绩，也不能用做任务的数量冒充能力。低负担入口能减少首次使用阻力，同时保留后续校准空间。

**约束：** 六题快速基线只覆盖阅读与词汇，不输出听力、口语或官方考试分数。推荐可使用画像排序并说明理由，但不得把推荐点击或 XP 直接写成能力提升。

## 2026-07-18 - 拉片采用用户授权素材和可组合训练

**决定：** 拉片保留为 `v0.9.0` 核心能力，先支持用户合法导入字幕与本地媒体，再逐步增加逐句循环、变速、自动暂停、听写、影子跟读、词块保存和考试难度出题。

**原因：** 欧路等成熟产品证明短片语境有吸引力，但固定片单和固定步骤容易僵化。Language Coach 应让用户自由开关字幕、翻译、提示和训练方式，并由画像控制难度。

**约束：** 不下载、打包或分发未经授权的视频和字幕；流媒体只在平台规则允许时通过浏览器上下文工作。观看本身不增加能力分，必须有可验证的听写、复述或作答证据。

## 2026-07-18 - 主导航和主从视图始终保持左右布局

**决定：** 主导航在桌面、平板和手机宽度都固定在左侧；文章、词典等标题列表与选中详情也保持左/右关系。窄屏通过缩窄侧栏和工作区横向滚动解决空间不足，不改成上下堆叠。

**原因：** 左侧功能位置稳定、标题与内容同时可见，是当前产品已确认的核心使用习惯。响应式适配不应改变信息架构和导航肌肉记忆。

**约束：** 内容区内部的表单、统计和解析模块仍可按宽度改为单列；固定左右指主导航以及具有明确主从关系的列表/详情视图。

## 2026-07-18 - 画像按七天周期和能力分项校准

**决定：** 训练画像每七天在完整 session 归档后检查。某分项至少需要 8 题、2 个 session 且跨 2 天才允许有限幅度调整；直接证据只更新对应分项。综合等级至少需要 3 个合格分项，并同时包含输入与输出能力。

**原因：** 单题或单日表现噪声较大，把阅读成绩直接外推到听说写会制造虚假提升。七天周期兼顾反馈速度和稳定性，分项隔离让推荐更可信。

**约束：** `weekly-rule-v1` 目前使用正确率、难度和信心错误，不使用尚未记录的单题时间或提示行为。XP、计划完成和单纯浏览不进入能力校准；算法输出不是正式考试预测。
